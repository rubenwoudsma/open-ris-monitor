# Roadmap

## Fase 1, structuur en ontwerp

- Repo-structuur vastleggen.
- README schrijven.
- Architectuur documenteren.
- Canoniek datamodel documenteren.
- Opslagbeleid documenteren.
- Huizen-config maken.

## Fase 2, eerste connector

- Base connector interface maken.
- GemeenteOplossingen connector maken.
- Meetings ophalen.
- Agendapunten ophalen.
- Documentmetadata ophalen.

## Fase 3, normalisatie

- Pydantic modellen implementeren.
- Stabiele identifiers genereren.
- JSONL exports maken.
- Schema-validatie toevoegen.

## Fase 4, verrijking

- Checksums berekenen.
- PDF's tijdelijk downloaden.
- Tekstextractie toevoegen.
- Kwaliteitsissues detecteren.
- Documentversies bijhouden.

## Fase 5, publicatie

- GitHub Actions workflow.
- Public data exports.
- GitHub Pages site.
- Simpele zoekfunctie.
- Downloadpagina.

## Fase 6, forkbaarheid

- Documentatie voor nieuwe gemeenten.
- Voorbeeldconfiguratie.
- Connectorcontract documenteren.
- Issue templates toevoegen.
